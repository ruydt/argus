## Direction findings (maintainer's call — not scheduled as plans)

Surfaced during the audit, grounded in repo evidence, left for the maintainer to weigh —
each could become a design/spike plan if chosen:

- **Collection/registry CRUD asymmetry** — add/publish/remove endpoints exist, but no
  *edit/update*; publishing a fix means hand-PRing on GitHub. Natural next step. (M)
- **Agent adapter as a named interface** — `agents/{claudecode,codex}` already implement
  the same 4-method contract; a registry + `Agent` interface would let a third agent (or
  community adapters) drop in without editing `hook.go` + frontend + presets in lockstep. (M-L)
- **Retention is env-only, default-off, no UI** — `ARGUS_RETENTION_DAYS`/`ARGUS_MAX_EVENTS`
  wire it (`main.go`), but with no sane default and no Diagnostics surface, DBs grow
  unbounded until disk fills. A default + a "delete before date" control productizes
  existing plumbing. (M)

## Findings considered and rejected (so they are not re-audited)

- **`ListSessionsByCWDPage` nil file-change counts** — non-finding; the code already
  guards `if countErr != nil { … } else { update }`, and a nil map read is a safe zero in
  Go (`event_service.go:817-824`).
- **`ListByTimeRange` pagination "infinite loop on empty page"** — non-finding; `hasMore`
  is computed from a `limit+1` fetch (`sqlite.go:307-317`), so an empty page returns
  `hasMore=false` and the `minID=0` cursor is never re-used.
- **`useLiveEvents` flush-on-cleanup / dedup-Set rebuild** — near-zero value; the effect
  resets `events` to `[]` on dep change anyway, and the dedup Set is capped at
  `MAX_LIVE_EVENTS=2000` (`useLiveEvents.ts`).
- **`usageScannedAt` check-then-act race / `sessionOutsideRange` string-compare fallback**
  — real but low value: the usage path is idempotent (`UpsertSession` ON CONFLICT), and
  the string fallback only triggers on malformed timestamps (`event_service.go:175-188,
  661-693`).
- **Dashboard multi-query / `DiagnosticsAgentStats` correlated subquery / `GetSessionTree`
  in-memory sort / count+data split** — marginal at local single-user SQLite scale;
  migration `017_audit_indexes.sql` already shipped the indexes that mattered.
- **`useSessions` polls all sessions every 5s** (`useSessions.ts:21,39`) — real at scale
  but the fix needs backend cursor pagination; medium effort, deferred. Reconsider if a
  user reports a large-session-list slowdown.
- **`FileSystemCard.tsx` / `StructuredEditor.tsx` god components (508 / 687 lines)** —
  risky churn for no behavior change, and `FileSystemCard` was just rewritten in this same
  branch; bad timing. Revisit when it next needs a feature change.
- **Reveal/CollectionReveal symlink TOCTOU, community-runtime allowlist** — low/no value;
  the allowlist guard is already present and correct.
