BEFORE (the pattern to find):
```go
_ = json.NewEncoder(w).Encode(v)
```

AFTER (the pattern to write, same indentation as surrounding code):
```go

    log.Printf("[handler] encode %T: %v", v, err)
}
```

Where `v` is whatever was being encoded (the second argument to Encode). The `%T` format verb prints the Go type of v, which is the intended debug signal per D-02.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Information Disclosure | log.Printf in encode error path | accept | The log line prints the Go type name (%T) and the error string (%v) — not the encoded payload content. No user data, no secrets, no PII is written to the log. The encode error itself is extremely unlikely in practice (would require http.ResponseWriter.Write to fail, e.g., broken pipe). Acceptable for a local-first developer tool. |
| T-07-03-02 | Denial of Service | encode error logging | accept | log.Printf is synchronous and blocking during request handling, but only on the error path. Error path requires a broken connection — attacker cannot trigger it without already having severed the TCP session. No amplification risk. |
</threat_model>
