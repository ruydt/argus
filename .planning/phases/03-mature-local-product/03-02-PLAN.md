<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| hook payload -> backend ingestion | Untrusted agent hook JSON enters the backend and may contain sensitive prompts, diffs, file paths, and tool output. |
| ignore file -> matcher | User-controlled ignore patterns affect whether events are persisted. |
| backend -> logs | Metadata leaves the request path into logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Information Disclosure | `handler.Hook` | mitigate | Apply ignore gate before `svc.AddEvent`, session update, and SSE broadcast per D-03. |
| T-03-02-02 | Information Disclosure | ignored-event logging | mitigate | Log only agent/session/action/reason metadata; do not log raw payload, prompt text, tool output, diffs, command, or path per D-04. |
| T-03-02-03 | Tampering | ignore pattern file | accept | Ignore file is intentionally user-controlled local policy; tests lock D-05 semantics to avoid accidental over/under matching. |
| T-03-02-04 | Denial of Service | unreadable ignore file | mitigate | Missing default file is empty matcher; unreadable explicit `HOOKER_IGNORE` returns actionable startup error. |
| T-03-02-SC | Tampering | Go module dependency | mitigate | Use dependency only if 03-01 approved exact `github.com/git-pkgs/gitignore@v1.2.0`; otherwise use scoped internal matcher. |
</threat_model>
