<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser origin -> local API | Browser requests from arbitrary origins attempt to read local sensitive API responses. |
| environment config -> network listener | `ADDR` and Docker defaults decide which network interfaces can reach hooker. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Information Disclosure | CORS middleware | mitigate | Exact allowlist; echo only matched origins; reject `null` and disallowed preflights; add `Vary: Origin`. |
| T-03-03-02 | Spoofing | CORS origin handling | mitigate | No wildcard and no arbitrary origin reflection; defaults derived from configured local port only. |
| T-03-03-03 | Information Disclosure | bind address validation | mitigate | Reject non-loopback `ADDR` unless `HOOKER_ALLOW_REMOTE=1` per D-07/D-08. |
| T-03-03-04 | Repudiation | remote bind startup | mitigate | Emit prominent `slog.Warn` warning with captured categories and unsupported public exposure per D-09. |
| T-03-03-05 | Configuration | Docker runtime | mitigate | Explicitly plan Docker behavior so `ADDR=0.0.0.0:8765` is not a silent bypass of the remote-bind gate. |
</threat_model>
