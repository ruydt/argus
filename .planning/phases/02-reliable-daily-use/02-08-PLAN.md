Add a `playwright` job to `.github/workflows/ci.yml`. The job must:
- `needs: [backend, frontend]` — runs after both jobs complete (binary must exist, frontend dist must exist)
- Use `ubuntu-latest`
- Set up Go (same version file as backend job)
- Set up Node.js 20
- Enable corepack + pnpm (same as other jobs)

Job steps:
```yaml
  playwright:
    runs-on: ubuntu-latest
    needs: [backend, frontend]
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v5
        with:
          go-version-file: backend/go.mod
          cache-dependency-path: backend/go.sum

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Setup pnpm via corepack
        run: corepack enable && corepack prepare pnpm@10.23.0 --activate

      - name: Build frontend (required for go:embed)
        working-directory: frontend
        run: |
          pnpm install --frozen-lockfile
          pnpm run build

      - name: Sync frontend dist into backend embed path
        run: |
          rm -rf backend/internal/ui/dist
          mkdir -p backend/internal/ui
          cp -R frontend/dist backend/internal/ui/dist

      - name: Build backend binary
        working-directory: backend
        run: go build -o hooker ./cmd/server

      - name: Install Playwright dependencies
        run: |
          npm install --prefix . @playwright/test 2>/dev/null || pnpm install --prefix . 2>/dev/null || true
          npx playwright install chromium --with-deps

      - name: Start hooker server
        run: |
          DB_PATH=/tmp/hooker-playwright-ci.db ./backend/hooker &
          until curl -sf http://127.0.0.1:8765/healthz; do sleep 1; done
        timeout-minutes: 1

      - name: Run Playwright smoke tests
        run: npx playwright test --project=chromium

      - name: Upload Playwright report on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 7
```

Note on `pnpm install --prefix .` vs npm: If root package.json was created with npm, use
`npm install` in CI. If it was created with pnpm, use `pnpm install`. The Playwright install
approach depends on what was done in Task 1 — check the actual root package.json and match.

Simplest approach for CI: just run `npm ci` or `npm install` at root (npm is always available
on ubuntu-latest without corepack). Use `npm install --save-dev @playwright/test` in the
install step regardless of local package manager.

Ensure the playwright job does NOT use `--frozen-lockfile` unless a lockfile exists at root.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Playwright test → running server | E2E test posts fixture data to a running server; test data goes through full ingest path |
| CI runner → backend binary | CI builds and starts the binary; uses temp SQLite file at /tmp |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | Playwright fixture data persisting between CI runs | accept | CI uses fresh /tmp/hooker-playwright-ci.db per job run; temp files are ephemeral on ubuntu-latest runners |
| T-02-08-02 | Denial of Service | Playwright server start timeout | mitigate | `until curl -sf healthz; do sleep 1; done` with `timeout-minutes: 1` ensures job fails fast if binary doesn't start |
| T-02-08-03 | Information Disclosure | Playwright test artifacts uploaded on failure | accept | Artifacts contain only screenshots/traces from the smoke test; no user data (fresh CI DB). Retention: 7 days. |
</threat_model>
