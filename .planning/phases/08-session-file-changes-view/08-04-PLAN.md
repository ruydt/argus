<diagnosis>
User reported: "i only see files changed displayed for claudecode but not codex".

Root cause:
- Codex edits commonly arrive as `apply_patch` tool events.
- `codex.Normalize` parses apply_patch hunks, but clears `OldString` and `NewString` to force patch rendering.
- `sqlite.GetFileChanges` only includes rows when the tool name is in `fileChangeCondition` or old/new strings are non-empty.
- `apply_patch` is not in `fileChangeCondition`, and command-only patch events do not contain `old_string`/`new_string`.
- `FileChangesList` renders `old_string`/`new_string` snippets, not raw patch commands.

Therefore Codex patch edits can be stored as trace events but omitted from `/api/file-changes`, so the Phase 8 page misses Codex file changes.
</diagnosis>
</context>
