# SearchableSelect — Design

**Date:** 2026-06-11
**Status:** Approved
**Scope:** Searchable select for long/growing option lists, adopted in the hooks-config simulator command picker.

## Decisions

- shadcn Combobox pattern: new `command` primitive (cmdk) composed with the existing `popover` primitive. Approach approved over in-Select search hacks (radix focus conflicts) and raw cmdk (style divergence).
- Adoption scope: simulator command picker ONLY. Event-type select, EventFilters action filter (12 static enums), StructuredEditor selects (short lists) stay plain `Select`.

## Components

### New primitive — `frontend/src/components/ui/command.tsx`

Generated by `npx shadcn@latest add command`. Never hand-edited. Adds `cmdk` dependency.

### New shared component — `frontend/src/components/shared/SearchableSelect.tsx`

- Props:
  ```ts
  type SearchableSelectOption = { label: string; value: string }
  type SearchableSelectProps = {
    value: string
    onValueChange: (v: string) => void
    options: SearchableSelectOption[]
    placeholder: string
    ariaLabel: string
    disabled?: boolean
    emptyText?: string // default "No results"
  }
  ```
- Render: Popover whose trigger is a button styled like the existing `SelectTrigger` (border, height, chevron icon, truncated selected-option label or muted placeholder). Content: `Command` with `CommandInput` (placeholder "Search…"), `CommandList`, `CommandEmpty` (emptyText), one `CommandItem` per option with a check icon when `option.value === value`.
- cmdk default fuzzy filtering on the item text.
- Selecting an item calls `onValueChange(option.value)` and closes the popover.
- Trigger carries `aria-label={ariaLabel}` and `role="combobox"` semantics (cmdk/popover provide the rest).

### Adoption — `frontend/src/features/hooks-config/SimulatorTab.tsx`

- Replace the second `Select` (command picker) with `SearchableSelect`:
  - `options` = existing `commandOptions` (config hooks + scripts, already deduped) + trailing `{ label: 'Custom command…', value: CUSTOM_VALUE }`
  - same `commandValue` / `handleCommandValueChange` wiring, same `disabled={!eventType}`, `ariaLabel="Select hook command"`
- The event-type `Select` is untouched.

## Testing

- New `frontend/tests/components/shared/SearchableSelect.test.tsx` (or alongside existing shared-component tests if a different directory exists): renders trigger with placeholder; opening shows all options; typing in the search input filters; clicking an option fires `onValueChange` and closes; empty search state shows emptyText.
- Update `frontend/tests/features/hooks-config/SimulatorTab.test.tsx`: interactions move from radix Select to the combobox (open popover, optionally type, click item). Reuse the existing jsdom stubs (scrollIntoView, pointer capture); add a ResizeObserver stub if cmdk requires it.

## Out of scope

- Converting any other Select usage.
- Multi-select, option groups, async loading.
